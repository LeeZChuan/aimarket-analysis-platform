# AI研判产品需求文档

更新时间：2026-04-28

## 1. 背景与目标

当前交易面板提供 K 线、股票列表和 AI 对话，但普通用户在问“某只股票走势如何”时，仍需要自己选择股票、选择分析角度、阅读较长回答。AI研判入口的目标是把这个流程压缩成一句话输入，并直接输出结论明确、重点突出的报告。

首版目标：

- 用户输入自然语言问题，例如“分析苹果走势”。
- 系统自动识别股票，生成短线或长线报告。
- 报告先给结论，再给三类关键数据红绿灯。
- 不伪造缺失数据；关键数据来自后端聚合接口。

## 2. 目标用户

- 不想配置复杂参数、只想快速看结论的普通投资者。
- 高频查看个股走势、需要短线判断依据的用户。
- 关注基本面、想看长期持有逻辑的用户。

## 3. 核心需求

### 3.1 独立入口

- 在全局导航新增 `AI研判`。
- 路由为 `/ai-report`。
- 该入口受现有登录保护控制。

### 3.2 首次偏好选择

- 用户第一次进入页面时，必须选择：
  - `短线`：只展示技术面、资金动向、明日涨跌概率。
  - `长线`：只展示估值、财报、行业前景。
- 选择后保存在本地，后续默认使用。
- 页面顶部允许用户随时切换偏好，切换后下一次生成报告使用新偏好。

### 3.3 自然语言输入

- 输入框支持股票代码、股票名称和自然语言问题。
- 示例：
  - “分析苹果走势”
  - “英伟达明天会涨吗”
  - “特斯拉适合长期拿吗”
- 前端把原始问题传给后端聚合接口，由后端完成股票识别和报告生成。
- 如果后端返回识别置信度偏低，前端展示最多 3 个候选股票，用户可点选候选并重跑报告。

### 3.4 报告结论

报告顶部必须先展示明确结论：

- `看涨`
- `观望`
- `看跌`
- `回避`

结论卡需要展示：

- 股票代码和名称。
- 结论标题。
- 简短摘要。
- 结论置信度。
- 红/黄/绿风险色。

### 3.5 三个关键红绿灯

报告必须包含三类信号：

- `汇率波动`
- `主力资金流向`
- `机构持仓变化`

每个信号展示：

- 状态：绿色=安全，黄色=关注，红色=危险。
- 当前数值。
- 一句话原因。
- 可选趋势线。

### 3.6 数据缺失策略

- 前端不自行编造汇率、主力资金、机构持仓数据。
- 后端返回 `dataQuality.missing` 时，前端显示“数据源暂缺”。
- 报告失败时展示错误和重试，不退化成假报告。

## 4. 非目标

首版不包含：

- 下单交易。
- 投资组合管理。
- 实时预警。
- 多股票组合对比。
- 前端自行抓取第三方数据源。

## 5. 成功标准

- 用户从进入页面到生成报告只需要选择一次偏好、输入一句话、点击一次按钮。
- 报告首屏能看到结论和三色风险灯。
- 短线和长线用户看到的重点内容不同。
- 后端接口不可用或数据缺失时，不展示虚假结论。
