# 动态设计

## 持续时间：100/300/500 法则

时机比缓动效果更重要。以下持续时间适用于大多数 UI：

| 持续时间 | 用例 | 示例 |

|----------|----------|----------|

| **100-150 毫秒** | 即时反馈 | 按钮按下、切换、颜色变化 |

| **200-300 毫秒** | 状态变化 | 菜单打开、工具提示、悬停状态 |

| **300-500 毫秒** | 布局变化 | 手风琴、模态框、抽屉 |

| **500-800 毫秒** | 进入动画 | 页面加载、主图展示 |

**退出动画比进入动画更快**——使用进入动画持续时间的约 75%。

## 缓动：选择合适的曲线

**不要使用 `ease`。** 这是一种折衷方案，很少能达到最佳效果。建议使用：

| 曲线 | 用途 | CSS |

|-------|---------|-----|

| **ease-out** | 元素进入 | `cubic-bezier(0.16, 1, 0.3, 1)` |

| **ease-in** | 元素离开 | `cubic-bezier(0.7, 0, 0.84, 0)` |

| **ease-in-out** | 状态切换（从进入 → 返回） | `cubic-bezier(0.65, 0, 0.35, 1)` |

**对于微交互，请使用指数曲线**——它们感觉自然，因为它们模拟了真实的物理现象（摩擦力、减速）：

```css

/* Quart 缓出 - 平滑、精致（推荐默认值）*/

--ease-out-quart: cubic-bezier(0.25, 1, 0.5, 1);

/* Quint 缓出 - 略微更具戏剧性 */

--ease-out-quint: cubic-bezier(0.22, 1, 0.36, 1);

/* Expo 缓出 - 干脆利落、自信满满 */

--ease-out-expo: cubic-bezier(0.16, 1, 0.3, 1);

```

**避免使用弹跳和弹性曲线。**它们在 2015 年很流行，但现在显得俗气且不专业。真实的物体在停止时不会弹跳——它们会平滑地减速。过冲效果会将注意力集中在动画本身，而不是内容上。

## 你应该设置动画的两个属性

**transform** 和 **opacity** 属性都可以设置动画——其他任何属性都会导致布局重新计算。对于高度动画（例如手风琴），请使用 `grid-template-rows: 0fr → 1fr` 而不是直接设置 `height` 属性的动画。

## 交错动画

使用 CSS 自定义属性可以实现更清晰的交错效果：`animation-delay: calc(var(--i, 0) * 50ms)`，并在每个项目上设置 `style="--i: 0"`。**限制总交错时间**——10 个项目，每个项目 50 毫秒，总共 500 毫秒。如果项目数量较多，请减少每个项目的延迟时间或限制交错项目的数量。

## 减少动态效果

这不是可选项。前庭功能障碍影响约 35% 的 40 岁以上成年人。

```css

/* 正常定义动画 */

.card {

animation: slide-up 500ms ease-out;

}

/* 提供减少动画效果的替代方案 */

@media (prefers-reduced-motion: reduce) {

.card {

animation: fade-in 200ms ease-out; /* 使用淡入淡出效果代替动态效果 */

}
}

/* 或者完全禁用动画 */

@media (prefers-reduced-motion: reduce) {

*, *::before, *::after {

animation-duration: 0.01ms !important;

transition-duration: 0.01ms !important;

}

}
```

**需要保留的内容**：进度条、加载指示器（速度减慢）和焦点指示器等功能性动画仍然有效，只是不包含空间移动效果。

## 感知性能

**没人关心你的网站实际运行速度有多快——他们只关心用户感觉有多快。** 感知性能与实际性能同样重要。

**80毫秒阈值**：我们的大脑会缓冲约80毫秒的感官输入以同步感知。任何低于80毫秒的操作都会让人感觉瞬间完成。这是微交互的目标。

**主动等待时间与被动等待时间**：被动等待（例如盯着加载指示器）比主动参与感觉更漫长。以下策略可以平衡这种差异：

- **抢占先机**：在加载过程中立即开始过渡（例如iOS应用缩放、骨架式UI）。用户会感觉到操作正在进行。

- **提前完成**：逐步显示内容——不要等待所有内容加载完毕。例如视频缓冲、渐进式图像加载、流式HTML加载。

- **乐观式UI**：立即更新界面，优雅地处理故障。Instagram点赞功能支持离线工作——UI立即更新，稍后同步。适用于低风险操作；避免用于支付或破坏性操作。

**缓动效果影响感知时长**：缓入（加速直至完成）会使任务感觉更短，因为峰终效应会显著增强最后阶段的效果。缓出在进入时会带来愉悦感，但缓入在任务结束时会压缩感知时间。

**注意**：响应过快会降低感知价值。用户可能不信任复杂操作（搜索、分析）的即时结果。有时，短暂的延迟表明“真正的工作”正在进行。

## 性能

不要提前使用 `will-change`——仅在动画即将开始时使用（`:hover`、`.animating`）。对于滚动触发的动画，请使用交叉观察器而不是滚动事件；动画播放一次后取消观察。创建运动标记以保持一致性（持续时间、缓动效果、常用过渡）。

---

**避免**：所有动画都使用（动画疲劳是真实存在的）。使用超过 500 毫秒的 UI 反馈。忽略 `prefers-reduced-motion`。使用动画来隐藏加载缓慢的问题。
