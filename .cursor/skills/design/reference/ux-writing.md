# 用户体验文案

## 按钮标签问题

**切勿使用“确定”、“提交”或“是/否”。** 这些标签显得敷衍且含义模糊。请使用特定的动词+宾语模式：

| 不好 | 好 | 为什么 |

|-----|------|-----|

| 确定 | 保存更改 | 说明将要发生的情况 |

| 提交 | 创建帐户 | 注重结果 |

| 是 | 删除消息 | 确认操作 |

| 取消 | 继续编辑 | 明确“取消”的含义 |

| 点击此处 | 下载 PDF | 描述目标位置 |

**对于破坏性操作**，请明确说明破坏行为：

- “删除”而非“移除”（删除是永久性的，移除意味着可恢复）

- “删除 5 个项目”而非“删除所选项目”（显示数量）

## 错误消息：公式

每个错误消息都应该回答以下问题：(1) 发生了什么？(2) 为什么？(3) 如何解决？示例：“电子邮件地址无效。请添加 @ 符号。”而不是“输入无效”。

### 错误消息模板

| 情况 | 模板 |

|-----------|----------|

| **格式错误** | “字段”需要使用 [格式]。例如：[示例]” |

| **缺少必填项** | “请输入 [缺少的内容]” |

| **权限不足** | “您没有访问 [内容] 的权限。[替代方案]” |

| **网络错误** | “我们无法连接到 [内容]。请检查您的连接并执行 [操作]。” |

| **服务器错误** | “我们的服务器出现故障。我们正在调查。[替代方案]” |

### 不要责怪用户

重新表述错误：“请以 MM/DD/YYYY 格式输入日期”，而不是“您输入的日期无效”。

## 空状态是机遇

空状态是引导用户入门的关键时刻：(1) 简要说明，(2) 解释填写空状态的价值，(3) 提供清晰的操作。“目前还没有项目。创建您的第一个项目即可开始。” 而不仅仅是“没有项目”。

## 语音与语气

**语音** 代表您品牌的个性——始终保持一致。

**语气** 则根据具体情况而变化。

| 时刻 | 语气转变 |

|--------|------------|

| 成功 | 庆祝式，简洁：“完成！您的更改已生效。” |

| 错误 | 同理心，乐于助人：“操作失败。您可以尝试以下方法……” |

| 正在加载 | 安抚人心：“正在保存您的工作……” |

| 删除确认 | 严肃，清晰：“删除此项目？此操作无法撤销。” |

**切勿用幽默的方式处理错误。** 用户已经感到沮丧了。要实用，不要卖萌。

## 无障碍写作

**链接文本**必须具有独立含义——例如“查看定价方案”，而不是“点击这里”。**替代文本**描述信息​​，而不是图像——例如“第四季度收入增长 40%”，而不是“图表”。装饰性图像请使用 `alt=""`。**图标按钮**需要 `aria-label` 以便屏幕阅读器识别上下文。

## 翻译写作

### 扩展规划

德语文本比英语文本长约 30%。分配空间：

| 语言 | 扩展 |

|----------|-----------|

| 德语 | +30% |

| 法语 | +20% |

| 芬兰语 | +30-40% |

| 中文 | -30%（字符数减少，但宽度相同） |

### 便于翻译的格式

数字要分开显示（例如“新消息：3”，而不是“您有 3 条新消息”）。使用完整的句子作为单个字符串（词序因语言而异）。避免使用缩写（例如，“5 分钟前”而不是“5 分钟前”）。向译者提供字符串出现的上下文。

## 一致性：术语问题

选择一个术语并坚持使用：

| 不一致 | 一致 |

|--------------|------------|

| 删除 / 移除 / 回收站 | 删除 |

| 设置 / 首选项 / 选项 | 设置 |

| 登录 / 登录 / 输入 | 登录 |

| 创建 / 添加 / 新建 | 创建 |

建立术语表并严格执行。术语过多会造成混乱。

## 避免冗余

如果标题已经解释清楚，那么引言就是多余的。如果按钮的含义很明确，就不要重复解释。只需说一遍，表达清楚即可。

## 加载状态

使用更具体的语言：“正在保存您的草稿……”而不是“正在加载……”。对于长时间的等待，请明确告知预期时间（例如“通常需要 30 秒”）或显示进度。

## 确认对话框：谨慎使用

大多数确认对话框都是设计缺陷——不妨考虑使用撤销功能。如果必须确认：请明确操作名称，解释后果，并使用具体的按钮标签（例如“删除项目”/“保留项目”，而不是“是”/“否”）。

## 表单说明

显示带有占位符的格式，而不是说明。对于不明显的字段，请解释询问原因。

---

**避免**：使用没有解释的术语。责怪用户（例如“您犯了一个错误”→“此字段为必填项”）。含糊不清的错误描述（例如“出了点问题”）。为了追求多样性而使用不同的术语。用幽默的方式来描述错误。
