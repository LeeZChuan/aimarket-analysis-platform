# 交互设计

## 八种交互状态

每个交互元素都需要设计以下状态：

| 状态 | 出现时间 | 视觉处理 |

|-------|------|------------------|

| **默认** | 静止状态 | 基本样式 |

| **悬停** | 指针悬停（非触摸）| 轻微抬升，颜色变化 |

| **聚焦** | 键盘/程序聚焦 | 可见圆环（见下文） |

| **激活** | 被按下 | 按下，颜色变深 |

| **禁用** | 不可交互 | 降低透明度，无指针 |

| **加载中** | 正在处理 | 旋转图标，骨架 |

| **错误** | 无效状态 | 红色边框，图标，消息 |

| **成功** | 已完成 | 绿色对勾，确认 |

**常见错误**：设计悬停状态时没有聚焦状态，反之亦然。它们是不同的。键盘用户通常看不到悬停状态。

## 焦点环：正确使用

**切勿直接使用 `outline: none` 而不进行替换。** 这违反了可访问性规范。请使用 `:focus-visible` 来仅对键盘用户显示焦点：

```css

/* 隐藏鼠标/触摸焦点环 */

button:focus {

outline: none;

}

/* 显示键盘焦点环 */

button:focus-visible {

outline: 2px solid var(--color-accent);

outline-offset: 2px;

}

```

**焦点环设计**：

- 高对比度（与相邻颜色对比度至少为 3:1）

- 2-3px 粗细

- 与元素保持一定距离（而不是位于元素内部）

- 在所有交互元素中保持一致

## 表单设计：一些不易察觉的细节

**占位符不是标签**——它们会在输入内容后消失。请始终使用可见的 `<label>` 元素。 **在失去焦点时进行验证**，而不是每次按键都进行验证（密码强度验证除外）。将错误信息**放在字段下方**，并使用 `aria-describedby` 属性连接字段。

## 加载状态

**乐观更新**：立即显示成功信息，失败时回滚。适用于低风险操作（例如点赞、关注），不适用于支付或破坏性操作。**骨架屏优于加载指示器**——骨架屏预览内容形状，速度比普通加载指示器更快。

## 模态框：惰性方法

过去，模态框中的焦点捕获需要复杂的 JavaScript 代码。现在可以使用 `inert` 属性：

```html

<!-- 当模态框打开时 -->

<main inert>

<!-- 模态框内的内容无法获得焦点或点击 -->

</main>

<dialog open>

<h2>模态框标题</h2>

<!-- 焦点保持在模态框内 -->

</dialog>

```

或者使用原生 `<dialog>` 元素：

```javascript

const dialog = document.querySelector('dialog');

dialog.showModal(); // 焦点陷阱打开，按 Esc 键关闭

```

## Popover API

对于工具提示、下拉菜单和非模态覆盖层，请使用原生 Popover：

```html

<button popovertarget="menu">打开菜单</button>

<div id="menu" popover>

<button>选项 1</button>

<button>选项 2</button>

</div>

```

**优点**：轻量级关闭（点击外部关闭）、正确的堆叠方式、避免 z-index 冲突、默认可访问。

## 可撤销操作：撤销 > 确认

**撤销比确认对话框更好**——用户会无意识地点击确认。立即从 UI 中移除，显示撤销提示，并在提示消失后真正删除。仅对真正不可逆的操作（例如删除帐户）、高成本操作或批量操作使用确认。

## 键盘导航模式

### 移动 Tab 键索引

对于组件组（标签页、菜单项、单选按钮组），每个组件组内只能有一个可使用 Tab 键的选项；方向键可在组件组内移动：

```html

<div role="tablist">

<button role="tab" tabindex="0">标签页 1</button>

<button role="tab" tabindex="-1">标签页 2</button>

<button role="tab" tabindex="-1">标签页 3</button>

</div>

```

方向键可在组件组之间移动 `tabindex="0"`。Tab 键则直接跳转到下一个组件。

### 跳转链接

为键盘用户提供跳转链接（`<a href="#main-content">跳转到主要内容</a>`），以便他们快速跳过导航。跳转链接在屏幕外隐藏，在获得焦点时显示。

## 手势识别

滑动删除等类似手势目前不可见。提示其存在：

- **部分显示**：从边缘露出删除按钮

- **引导**：首次使用时提供引导标记

- **替代方案**：始终提供可见的备用方案（包含“删除”的菜单）

不要依赖手势作为执行操作的唯一方式。

---

**避免**：移除焦点指示器而没有提供替代方案。使用占位符文本作为标签。触摸目标小于 44x44 像素。通用错误消息。不支持 ARIA/键盘的自定义控件。
