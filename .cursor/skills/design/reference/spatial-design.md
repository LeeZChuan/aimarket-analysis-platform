# 空间设计

## 间距系统

### 使用 4pt 基准值，而非 8pt

8pt 系统过于粗略——您经常需要 12px（介于 8 和 16 之间）。使用 4pt 可获得更精细的粒度：4、8、12、16、24、32、48、64、96px。

### 语义化命名标记

根据关系命名（`--space-sm`、`--space-lg`），而不是值（`--spacing-8`）。使用 `gap` 而不是 `margin` 来设置兄弟元素之间的间距——这样可以避免边距折叠和清理技巧。

## 网格系统

### 自适应网格

使用 `repeat(auto-fit, minmax(280px, 1fr))` 来实现无断点的响应式网格。列宽至少为 280 像素，每行列数应尽可能多，剩余部分可拉伸。对于复杂的布局，请使用命名网格区域（`grid-template-areas`），并在断点处重新定义它们。

## 视觉层级

### 眯眼测试

眯起眼睛（或截屏并模糊处理）。你还能识别出：

- 最重要的元素？

- 第二重要的元素？

- 清晰的分组？

如果所有元素在模糊处理后看起来粗细相同，则说明你的层级结构存在问题。

### 多维度层级

不要仅仅依赖大小。结合以下因素：

| 工具 | 强层级 | 弱层级 |

|------|------------------|----------------|

| **大小** | 3:1 或更大的比例 | 小于 2:1 的比例 |

| **粗细** | 粗体 vs 常规 | 中等 vs 常规 |

| **颜色** | 高对比度 | 相似色调 |

| **位置** | 左上（主要） | 右下 |

| **间距** | 留白充足 | 拥挤 |

**最佳层级结构同时运用 2-3 个维度**：标题更大、更粗，并且上方留有更多空间。

### 卡片并非必需

卡片已被过度使用。间距和对齐方式可以自然地创建视觉分组。仅当内容真正独特且可操作、需要在网格中对项目进行视觉比较，或内容需要清晰的交互边界时才使用卡片。**切勿在卡片内嵌套卡片**——使用间距、字体和巧妙的分隔符来创建卡片内的层级结构。

## 容器查询

视口查询用于页面布局。**容器查询用于组件**：

```css

.card-container {

container-type: inline-size;

}

.card {

display: grid;

gap: var(--space-md);

}

/* 卡片布局根据其容器而非视口进行调整 */

@container (min-width: 400px) {

.card {

grid-template-columns: 120px 1fr;

}

```

**为什么这很重要**：在较窄的侧边栏中，卡片保持紧凑布局；而在主内容区域中，同一张卡片会自动展开，无需任何视口调整。

## 视觉调整

`margin-left: 0` 处的文本由于字母间距而显得缩进——使用负边距（`-0.05em`）进行视觉对齐。几何居中的图标通常看起来偏离中心；播放图标需要向右移动，箭头需要向其方向移动。

### 触摸目标与视觉尺寸

按钮看起来可能很小，但需要较大的触摸目标（至少 44px）。使用内边距或伪元素：

```css

.icon-button {

width: 24px; /* 视觉尺寸 */

height: 24px;

position: relative;

}

.icon-button::before {

content: '';

position: absolute;

inset: -10px; /* 点击目标区域扩展至 44px */

}

```

## 深度和高度

创建语义化的 z-index 比例尺（下拉菜单 → 固定图标 → 模态框背景 → 模态框 → 提示框 → 工具提示），而不是使用任意数值。对于阴影，创建一致的高度比例尺（sm → md → lg → xl）。**关键要点**：阴影应该柔和——如果阴影清晰可见，则可能过强。

---

**避免**：在比例尺之外使用任意间距值。将所有间距设为相等（多样性有助于创建层次结构）。仅通过大小来创建层次感——结合大小、重量、颜色和空间。
