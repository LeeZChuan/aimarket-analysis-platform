# 色彩与对比度

## 色彩空间：使用 OKLCH

**停止使用 HSL。** 请改用 OKLCH（或 LCH）。它在感知上是均匀的，这意味着亮度级别的相同变化看起来*相同*——不像 HSL，50% 的黄色看起来很亮，而 50% 的蓝色看起来很暗。

```css
/* OKLCH：亮度 (0-100%)，色度 (0-0.4+)，色调 (0-360) */

--color-primary: oklch(60% 0.15 250); /* 蓝色 */

--color-primary-light: oklch(85% 0.08 250); /* 相同色调，更亮 */

--color-primary-dark: oklch(35% 0.12 250); /* 同色系，更深 */

```

**关键要点**：颜色越接近白色或黑色，饱和度（色度）越低。极高的饱和度在极亮的亮度下会显得刺眼。例如，亮度为 85% 的浅蓝色需要约 0.08 的饱和度，而不是基色的 0.15。

## 构建功能性调色板

### 带色调的中性色陷阱

**纯灰色毫无特色。** 为所有中性色添加一丝品牌色调：

```css

/* 死灰 */

--gray-100: oklch(95% 0 0); /* 缺乏个性 */

--gray-900: oklch(15% 0 0);

/* 暖色调灰色（增添品牌暖意） */

--gray-100: oklch(95% 0.01 60); /* 略带暖色调 */

--gray-900: oklch(15% 0.01 60);

/* 冷色调灰色（科技、专业） */

--gray-100: oklch(95% 0.01 250); /* 略带蓝色调 */

--gray-900: oklch(15% 0.01 250);

```

色度值虽然很小（0.01），但仍然可感知。它能在潜意识中将品牌颜色与用户界面联系起来。

### 调色板结构

一个完整的系统需要：

| 角色 | 用途 | 示例 |

|------|---------|---------|

| **主色** | 品牌、行动号召、关键操作 | 1 种颜色，3-5 种色调 |

| **中性色** | 文本、背景、边框 | 9-11 色阶 |

| **语义** | 成功、错误、警告、信息 | 4 种颜色，每种颜色 2-3 种色调 |

| **表面** | 卡片、模态框、叠加层 | 2-3 个高度级别 |

**除非必要，否则请跳过辅助/三级颜色。** 大多数应用只需一种强调色即可。添加更多强调色会造成决策疲劳和视觉干扰。

### 60-30-10 法则（正确应用）

此法则关注的是**视觉权重**，而非像素数量：

- **60%**：中性背景、留白、基础表面

- **30%**：辅助色——文本、边框、非激活状态

- **10%**：强调色——CTA、高亮显示、聚焦状态

常见错误：到处使用强调色，因为它是“品牌色”。强调色之所以有效，*正是*因为它们很少出现。过度使用会削弱其效果。

## 对比度和可访问性

### WCAG 要求

| 内容类型 | AA 最低标准 | AAA 目标 |

|--------------|------------|------------|

| 正文 | 4.5:1 | 7:1 |

| 大号文本（18px+ 或 14px 粗体） | 3:1 | 4.5:1 |

| UI 组件、图标 | 3:1 | 4.5:1 |

| 非必要装饰 | 无 | 无 |

**注意**：占位符文本仍需使用 4.5:1 的对比度。你随处可见的浅灰色占位符文本？通常不符合 WCAG 标准。

### 危险的颜色组合

以下颜色组合通常对比度不足或导致可读性问题：

- 白色背景上的浅灰色文本（最常见的可访问性问题）

- **任何颜色背景上的灰色文本**——灰色看起来会褪色且颜色不鲜艳。使用比背景色更深的色调，或使用透明度

- 红色文字配绿色背景（反之亦然）——8% 的男性无法区分这两种颜色

- 蓝色文字配红色背景（视觉上会产生抖动）

- 黄色文字配白色背景（几乎总是失败）

- 图片上的细浅文字（对比度难以预测）

### 切勿使用纯灰色或纯黑色

纯灰色 (`oklch(50% 0 0)`) 和纯黑色 (`#000`) 在自然界中并不存在——真实的阴影和表面总是带有色调。即使色度只有 0.005-0.01，也足以营造自然感，而不会显得过于明显。（参见上文的着色中性色示例。）

### 测试

不要相信你的眼睛。使用以下工具：

- [WebAIM 对比度检查器](https://webaim.org/resources/contrastchecker/)

- 浏览器开发者工具 → 渲染 → 模拟视觉缺陷

- [Polypane](https://polypane.app/) 用于实时测试

## 主题：浅色模式和深色模式

### 深色模式并非浅色模式的反转

您不能简单地交换颜色。深色模式需要不同的设计决策：

| 浅色模式 | 深色模式 |

|------------|-----------|

| 使用阴影来增强深度 | 使用较浅的表面来增强深度（无阴影） |

| 深色文字配浅色背景 | 浅色文字配深色背景（减小字体粗细） |

| 鲜艳的强调色 | 略微降低强调色的饱和度 |

| 白色背景 | 切勿使用纯黑色——使用深灰色（饱和度 12-18%） |

```css

/* 通过表面颜色而非阴影实现深色模式的深度 */

:root[data-theme="dark"] {

--surface-1: oklch(15% 0.01 250);

--surface-2: oklch(20% 0.01 250); /* “更高” = 更浅 */

--surface-3: oklch(25% 0.01 250);

/* 略微降低文本粗细 */

--body-weight: 350; /* 而不是 400 */

}
```

### 标记层级

使用两层：基本标记（`--blue-500`）和语义标记（`--color-primary: var(--blue-500)`）。对于深色模式，只需重新定义语义层——基本标记保持不变。

## Alpha 通道是一种设计缺陷

大量使用透明度（rgba、hsla）通常意味着调色板不完整。Alpha 通道会造成对比度不稳定、性能开销增加以及色彩不一致。建议为每个场景定义明确的叠加颜色。例外情况：焦点。需要透明效果的环形图案和交互式场景。

---

**避免**：仅依赖颜色来传达信息。创建调色板时未明确每种颜色的作用。在大面积区域使用纯黑色（#000）。忽略色盲测试（8% 的男性患有色盲）。
